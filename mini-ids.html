Overview

The Mini Intrusion Detection System is a hardware pattern-matching engine implemented via schematic capture in Xilinx ECS, targeting the NetFPGA-v2 platform (Virtex-2 Pro). It performs inline inspection of a live packet data stream, comparing each byte window against a stored signature pattern without interrupting throughput.

The design uses a sliding-window architecture: a 9-byte pipeline register (reg9B) retains the tail of the previous cycle's data, which is merged with the current 8-byte pipe1 stream to form a 14-byte (112-bit) window. Seven 56-bit comparator instances scan every possible alignment of the 7-byte pattern within that window, asserting a match signal if any position matches — subject to a 7-bit wildcard mask (AMASK). A matched packet is held in a 19-byte-wide dual-port block-RAM FIFO (dropfifo) until it can be flagged or dropped.

All Verilog netlists were auto-generated from ECS schematics, synthesized, and validated in simulation with Perl-generated traffic covering boundary cases.

Match register (match_en gated)
  • reg9B — Stores the 9 trailing bytes of the previous pipe1 cycle using a banked chain of FD8CE and FD16CE registers (1×8-bit + 4×16-bit), all sharing the same CE and CLR. Its output (pipe0) feeds busmerge alongside the current pipe1.
  • busmerge — Concatenates pipe0[47:0] (6 bytes from the previous word) with pipe1[63:0] (8 current bytes) to produce a 112-bit window (datain[111:0]), ensuring no gap where a 7-byte pattern could straddle a word boundary.
  • wordmatch — Instantiates 7 comparator modules, each checking a different byte offset of datain[111:0] against datacomp[55:0] (the 7-byte pattern from hwregA). AMASK[6:0] masks out don't-care bytes; any byte whose mask bit is set is excluded from the equality check. All 7 match outputs are OR'd together — if any position matches, the match line is asserted.
  • detect7B output register — An AND3B1 gate combines the raw match signal with match_en and the inverted current-match state. If match_en is high and no match is already latched, the new match is stored into the FDCE output register and driven out as the match signal.

Module Breakdown

comparator.sch
Seven COMP8 instances chained to form a full 56-bit equality comparator. Each COMP8 checks 8 bits of the data window against the pattern, gated by one AMASK bit. Results are AND-reduced; also reused in dropfifo to compare read/write FIFO pointers.
detect7B.sch
Top-level pattern-match engine. Instantiates reg9B, busmerge, wordmatch, and the FDCE output register. Connects hwregA (pattern), match_en, mrst, pipe1, clk, and CE. The AND3B1 gate ensures match is only latched when enabled and not already active.
reg9B.sch
A 9-byte (72-bit) pipeline register implemented as one FD8CE (bits 71:64) and four FD16CE blocks (bits 63:48, 47:32, 31:16, 15:0). All share CE and CLR, clocked together to hold the previous pipe1 word for the busmerge sliding window.
wordmatch.sch
Instantiates 7 comparator modules, each receiving a different 56-bit slice of the 112-bit datain window (offsets 0, 8, 16, 24, 32, 40, 48 bits). All share the same datacomp[55:0] pattern and wildcard[6:0] mask. Outputs are OR8'd into a single match line.
dropfifo.sch
19-byte-wide dual-port block RAM FIFO (dual19Bmem) with CB8CLE/CB8CE 8-bit counters for read/write address management. Two COMP8 instances compare addresses; if equal, the write register is not enabled, preventing overwrite. FDC outputs valid_data when a matched packet entry is ready.

Design Entry Note — All modules were designed as ECS schematics in Xilinx ISE. The Verilog netlists (e.g. detect78.v, wordmatch.v) were auto-generated by ECS and are not intended for manual editing per the tool's synthesis directives.
