Overview

  • Full-custom 16-bit MAC circuit designed in Cadence Virtuoso using the GPDK 045nm PDK, covering the complete IC design flow from schematic entry through post-layout parasitic extraction and simulation.
  • The multiplier datapath uses radix-4 Booth encoding to generate partial products, reduced through a tree of 3:2 and 6:2 carry-save compressors before a final 16-bit Kogge-Stone adder produces the result.
  • Sequential elements (DFF and DFF_RST) were designed from scratch using transmission-gate latches, with layout and full DRC/LVS sign-off on each block.
  • Gate sizing was swept across 1×, 2×, and 4× combinations to minimise propagation delay on the critical cin→S and cin→Cout paths.

LAB 03 1-bit Full Adder — Gate Sizing & Layout ＋

Schematic

  • XOR₄ₓ → XOR₁ₓ for the sum output; NAND carry tree optimised for the cin→S path. A second variant re-sizes the carry gates (NAND₂ₓ + NAND₄ₓ) to target the cin→Cout path instead.
  • Gate sizing swept 1×/2×/4× for both carry stages; best cin→S falling at 56.78 ps (4×/1×), best cin→Cout falling at 49.09 ps (2×/4×).

PHASE 1 Sequential Elements & 16-bit KSA Adder — DFF, DFF_RST, Kogge-Stone ＋

D Flip-Flop (DFF)

  • Master-slave topology using transmission-gate latches and tristate inverters; clock inverted once at top level to drive both halves.
  • DRC: 0 errors across 562 rule checks. LVS mismatch traced to an extraction issue unrelated to the core schematic.

DFF with Asynchronous Reset (DFF_RST)

  • Active-low async reset added via a NAND gate at the master latch input. Asserting RST_b = 0 forces Q → 0 independent of the clock edge.
DFF_RST Schematic NAND gate at master latch input gates RST_b signal
DFF_RST Functionality out_Q, in_D, clk, RST_b — async reset forces Q low on RST_b = 0

16-bit Kogge-Stone Adder

  • Parallel-prefix tree: 3 black-cell levels (NAND→INV, OR→INV) + 1 grey-cell level with final XOR sum. Worst-case path: NAND/INV + 4(NAND+INV) + XOR, exercised at A=0x0001, B=0xFFFF.

PHASE 2 Partial Product Reduction — Booth Encoder, 3:2 & 6:2 Compressors ＋

Radix-4 Booth Encoder

  • Encodes X<2:0> into a signed partial product PP<8:0> of Y<7:0>, generating {0, ±Y, ±2Y}. Implemented with XOR, NOR, MUX, and inverter primitives.
  • DRC: 0 errors (1335 geometries, 562 rules). LVS: MATCH. Post-extraction blocked by apostrophe characters in wire names inside XOR/MUX subcells.
Booth Encoder Schematic X<2:0>, Y<7:0>, zero<8:0> → PP<8:0>; XOR/NOR/MUX selection logic
Booth Encoder Layout DRC: 0 errors; LVS: MATCH
Booth Encoder — Pre-extraction Functionality Signed decimal annotations on X, Y, PP confirm ±Y and ±2Y encoding across all 8 X values

3:2 Compressor

  • Reduces three 16-bit inputs A, B, C to two outputs out0 + out1 using 16 FA_1b instances. DRC: 0 errors. LVS: MATCH. Post-extraction blocked by same apostrophe issue.
3:2 Compressor Schematic A, B, C<15:0> → out0, out1<15:0> via 16× FA_1b + bias circuit
3:2 Compressor Layout Stacked column of 16 FA_1b cells; DRC: 0 errors (656 geometries)
3:2 Compressor — Pre-extraction Functionality A, B, C, out0, out1 hex values; out0 + out1 = A + B + C confirmed for all 6 test vectors

6:2 Compressor

  • Cascades three 3:2 Encoder instances to reduce six 16-bit inputs A–F to two outputs. Pre-extraction simulation confirmed correct signed decimal sums for all 6 test vectors.
6:2 Compressor Schematic Three cascaded 3:2 Encoders; A–F<15:0> → out0, out1<15:0>
6:2 Compressor Layout Two parallel columns of stacked FA_1b cells across three compressor stages
