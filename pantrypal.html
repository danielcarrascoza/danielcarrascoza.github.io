Project Evolution

V1

JavaFX Desktop Application

PantryPal began as a standalone JavaFX desktop app with local user authentication and persistent recipe memory backed by MongoDB. Users could log in, track pantry ingredients, and retrieve saved recipes across sessions. AI recipe generation was integrated via LLM API calls triggered from the desktop UI.

V2

Full-Stack Web App on Firebase

The project was rebuilt and extended as a production web application, migrating infrastructure to Firebase (Hosting, Firestore, Auth) and rewriting the frontend in JavaScript. The backend engineering role covered Firebase architecture, real-time data sync, and coordinating cross-team integration across a multi-developer Agile team.

Architecture (ACTS)

System Architecture — Web Version
Browser / JS Frontend → Firebase Auth + Firestore DB
User Input (pantry items) → LLM API → AI-Generated Recipe
GitHub Repo → GitHub Actions (CI/CD) → Firebase Hosting
Firebase Hosting serves the static JS frontend with global CDN delivery
Firestore provides real-time NoSQL storage for user pantry state and saved recipes
Firebase Auth handles session management and user identity across devices
LLM API call triggered client-side on pantry submission → recipe streamed to UI

CI/CD Pipeline

GitHub Actions

Automated test and deploy pipeline triggered on every push to main. On merge, Actions runs linting and unit checks, then invokes the Firebase CLI to deploy the updated build directly to Firebase Hosting — zero manual deployment steps.

Firebase Integration

Firebase project config is managed via environment secrets in the Actions runner. Each deploy is atomic — Firebase rolls back automatically on hosting errors. Preview channels were used for feature-branch QA before merging to production.

Agile Process

  • Worked in a multi-developer team across structured Agile sprints — backlog grooming, sprint planning, and retrospectives coordinated via GitHub Projects and issues.
  • Backend engineering role covered Firebase infrastructure ownership: schema design in Firestore, Auth rule configuration, and unblocking frontend developers during integration milestones.
  • Cross-team coordination ensured the LLM API integration, Auth flow, and recipe storage were developed in parallel with minimal merge conflicts — enabled by clear interface contracts agreed on during sprint planning.

Key Features

  • AI recipe generation — users input available pantry ingredients and receive a full recipe generated by an LLM API, surfaced directly in the UI without leaving the app.
  • Persistent user accounts with Firebase Auth; pantry state and saved recipes sync to Firestore in real time and persist across sessions and devices.
  • Evolved from a MongoDB-backed JavaFX prototype into a cloud-native web app — the migration preserved the core user model while gaining scalability, real-time sync, and zero-infrastructure maintenance.
