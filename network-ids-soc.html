01 — System Architecture

Single-die data & control planes

The SoC is partitioned into two non-intersecting planes. The data plane handles every ingress byte: packets arrive at the 256×72 BRAM convertible FIFO, the DMA engine streams extracted flow features at 1 word/cycle into the SIMD coprocessor, and the verdict gates egress — the host ARM core never touches this path.

The control plane runs a 4-thread ARM-style CPU that polls FIFO status, configures DMA addresses, starts the kernel, and polls completion — pure orchestration. The coprocessor itself runs a CUDA-compatible custom ISA over a 4-lane BF16 + INT16 SIMD unit.

02 — ML Model

What actually runs on silicon

A 4-layer ReLU MLP (11 → 16 → 8 → 2) trained on KDDTrain+ (125,973 samples) with label smoothing 0.1. Inputs are the top-11 RF-importance-ranked flow features from 41 NSL-KDD attributes — mostly byte counts, service rates, and protocol flags. BF16 inference runs on-chip; label smoothing 0.1 was the single change that unlocked generalization on KDDTest+.

03 — BCAST Instruction

Cross-lane SIMD in one cycle

Without a cross-lane broadcast, reducing across 4 SIMD lanes requires N serial passes through the register file — 4 sequential cycles for 4 lanes. The BCAST instruction solves this: one instruction broadcasts a value across all lanes simultaneously in a single cycle. Measured using on-chip PCI cycle counters across 5/5 silicon runs, the speedup is a consistent 3.28×.

05 — Power Analysis

FPGA measured · ASIC projected

Xilinx XPower measured total chip power at 2,674 mW on the 90 nm NetFPGA. Using Kuon & Rose TCAD '07 scaling with a +6 dynamic gap adjustment for BRAM/mult-blended logic, the 28 nm ASIC projection lands at ~90–110 mW — three process generations of V²+C reduction. 7 nm projects a further 2–3× reduction to ~30–50 mW.
