Overview

  • Implemented CUDA kernels in C++ and exposed them to Python as a shared library via ctypes — enabling GPU-accelerated compute from a pure Python interface without any framework overhead.
  • Part 1: benchmarked naïve CUDA, shared-memory tiled matrix multiplication, and cuBLAS against a CPU baseline across matrix sizes N = 512 to 8192.
  • Part 2: implemented a custom GPU Sobel edge-detection filter compiled into a DLL, callable directly from Python with NumPy array inputs.

Hardware

CPU Baseline

Intel Core i7-9700K @ 3.6 GHz. Plain C matrix multiplication (matrix_cpu.c). Averaged over 5 runs per matrix size.

GPU Target

NVIDIA RTX 2070 Super. Naïve kernel (matrix_cuda.cu), optimized tiled kernel (matrix_cuda_opt.cu), and cuBLAS reference. All CUDA runs averaged over 5 iterations.

Analysis

  • GPU advantage is negligible at small sizes (N=512, ~1.1× speedup) due to kernel launch overhead — the GPU begins to significantly outperform the CPU at N=1024 (~6× speedup) and dominates at N=2048 with over 117× speedup for the naïve kernel.
  • Shared-memory tiling provides a modest but consistent improvement over the naïve kernel (121× vs. 117× at N=2048), with the gap widening further at N=4096 and N=8192.
  • cuBLAS carries higher startup overhead than both hand-written kernels at small N, but scales nearly flat — at N=8192 it is 3.75× faster than the optimized kernel, reflecting superior memory hierarchy utilization (likely tensor core and warp-level optimizations not accessible in hand-written CUDA).
  • Practical recommendation: hand-written tiled kernels are competitive for small-to-mid matrices; cuBLAS is the correct choice once N exceeds ~2048.

Part 2 — GPU Sobel Edge Detection

  • Implemented a CUDA Sobel filter (conv-sobel.cu) that computes horizontal and vertical gradient magnitudes in parallel across all pixels, compiled into a shared library (sobel_lib.dll).
  • Python wrapper (conv-sobel-called.py) loads the DLL via ctypes, passes NumPy uint8 arrays with C_CONTIGUOUS layout directly to the GPU kernel, and writes the edge-detected grayscale output to disk.
  • Validated on three test images (259×194, 277×182, 225×225) — clean edge maps with strong structural detail preserved across varied image content.
